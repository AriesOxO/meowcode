# 零依赖 AI Agent 可行性分析

## 核心问题

能否实现一个**几乎零外部依赖**的编码 Agent？

## 依赖分层

### Level 0: 绝对必需（1个）

```
只需要一个 LLM SDK
├── anthropic  (推荐，API 简洁)
├── openai     (生态最大)
└── zhipuai    (国内访问稳定)
```

**无法避免的原因**：必须与 LLM 通信，自己实现 HTTP 客户端没意义。

### Level 1: Python 标准库可替代

| 功能 | 常见依赖 | 标准库替代 |
|------|---------|-----------|
| 文件搜索 | ripgrep | `os.walk()` + `fnmatch` |
| 代码解析 | tree-sitter | 正则表达式 + 简单状态机 |
| 终端美化 | rich | `print()` + ANSI 转义码 |
| 配置管理 | pydantic | `json` + 手动验证 |
| HTTP 请求 | requests | `urllib.request` |

### Level 2: 可以完全不要

- ❌ 数据库（SQLite 也是标准库）
- ❌ Web 框架（如果不做 Web UI）
- ❌ 测试框架（用 assert 就行）

## 极简实现方案

### 方案 A：单文件 Agent（推荐入门）

```python
# agent.py - 只依赖 anthropic
import re
import os
import subprocess
import json
from anthropic import Anthropic

# 300 行以内实现所有核心功能
```

**优点**：
- 极易理解和调试
- 部署简单（1个文件 + pip install anthropic）
- 适合学习和快速原型

**能做到**：
- ✅ 读写文件
- ✅ 执行命令
- ✅ 简单代码搜索（正则）
- ✅ 对话循环
- ✅ 基础错误处理

**做不到**：
- ❌ 精确 AST 级别的代码修改
- ❌ 复杂的语法高亮
- ❌ 高性能全文搜索（但够用）

### 方案 B：标准库 + 1个 SDK（推荐实用）

依赖清单：
```txt
anthropic==0.40.0
```

就这一行！

核心实现：
```python
# 文件搜索 - 标准库版
def search_files(pattern: str, path: str = ".") -> list[str]:
    import os, fnmatch
    matches = []
    for root, dirs, files in os.walk(path):
        # 排除常见忽略目录
        dirs[:] = [d for d in dirs if d not in {'.git', 'node_modules', '__pycache__'}]
        for filename in files:
            if fnmatch.fnmatch(filename, pattern):
                matches.append(os.path.join(root, filename))
    return matches

# 代码搜索 - 正则版
def grep_content(pattern: str, file_path: str) -> list[tuple[int, str]]:
    import re
    results = []
    with open(file_path, 'r', encoding='utf-8', errors='ignore') as f:
        for i, line in enumerate(f, 1):
            if re.search(pattern, line):
                results.append((i, line.rstrip()))
    return results

# 精确编辑 - 字符串匹配版（够用）
def replace_in_file(file_path: str, old: str, new: str) -> bool:
    with open(file_path, 'r', encoding='utf-8') as f:
        content = f.read()
    
    if old not in content:
        return False
    
    content = content.replace(old, new, 1)  # 只替换第一次出现
    
    with open(file_path, 'w', encoding='utf-8') as f:
        f.write(content)
    return True

# 命令执行 - subprocess 版
def run_command(cmd: str, timeout: int = 30) -> dict:
    import subprocess
    result = subprocess.run(
        cmd,
        shell=True,
        text=True,
        capture_output=True,
        timeout=timeout
    )
    return {
        'stdout': result.stdout,
        'stderr': result.stderr,
        'returncode': result.returncode
    }
```

**这些功能足够实现 80% 的编码 Agent 能力！**

### 方案 C：渐进增强（推荐长期）

阶段 1（第 1 周）：
```
只依赖 anthropic
实现基础循环和文件操作
```

阶段 2（第 2-3 周）：
```
发现瓶颈后按需添加
比如搜索太慢才考虑 ripgrep
```

阶段 3（第 4+ 周）：
```
根据实际需求决定
需要语法树才加 tree-sitter
```

## 对比：依赖数量 vs 功能

| 依赖数 | 能实现的功能 | 适用场景 |
|--------|-------------|---------|
| 1 个 | 基础对话 + 命令执行 + 简单文件操作 | 学习原型 |
| 1-2 个 | + 代码搜索 + 精确编辑 | 个人工具 |
| 3-5 个 | + AST 解析 + 高性能搜索 | 团队使用 |
| 10+ 个 | + Web UI + 多 Agent + 插件系统 | 商业产品 |

## 我的建议

**从方案 A 开始**：单文件 + 只依赖 anthropic

理由：
1. **快速验证想法**：1-2 天就能跑起来
2. **深入理解本质**：不被工具链遮蔽核心逻辑
3. **容易迭代**：发现瓶颈时自然知道该加什么依赖
4. **博客素材丰富**：记录"为什么最后还是加了 XX 依赖"

**56 行代码就实现了可工作的 Agent！**

## 结论

✅ **可以做到几乎零依赖**（只要 1 个 LLM SDK）  
✅ **标准库功能足够强大**（Python 3.10+ 更是如此）  
✅ **按需添加依赖才是最佳实践**（不要过度设计）
